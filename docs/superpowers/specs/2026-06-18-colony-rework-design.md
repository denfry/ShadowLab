# Colony Survival → RimWorld-lite: дизайн переработки

- **Дата:** 2026-06-18
- **Игра:** `colony` (в коде — «Colony Evolution»)
- **Статус:** дизайн одобрен; ожидает ревью спеки перед планом реализации
- **Подход:** C — чистая архитектура систем + поставка играбельными фазами

---

## 1. Контекст и проблема

Текущая игра (`src/games/colony/`) ощущается слишком простой:

- Карта 24×24 с типами тайлов, но **террейн чисто косметический** — ни на что не влияет.
- 3 ресурса (еда/дерево/наука), 3 здания (ферма/дом/лаб), 3 технологии.
- Работы — **абстрактные счётчики**: колонисты не ходят в здания, безличны, без черт/навыков.
- Здания **ставятся автоматически** (`findBuildTile`) — нет пространственной стратегии.
- ~11 событий = «±N к ресурсу».
- Победа на день 20 при населении 15 — короткая петля; рейды = один бросок кубика.

**Цель переработки** — превратить игру в браузерный **RimWorld-lite**: глубина в людях и кризисах выживания.

## 2. Итоговое видение

> Полноценные колонисты-личности (имена, черты, навыки, личные потребности, травмы/болезни,
> психические срывы) на реальной сетке с поиском пути; комнаты с распространением тепла;
> четыре системы давления — климат, здоровье, рейды+оборона, психика+социум; длинный арк
> ~40–60 дней с эскалацией и финальной целью.

## 3. Принятые решения (лог брейншторминга)

| # | Вопрос | Решение |
|---|--------|---------|
| 1 | Направление углубления | Выживание и драма колонистов (RimWorld/Frostpunk) |
| 2 | Глубина колонистов | Полноценные личности: черты, навыки, нужды, травмы/болезни, срывы |
| 3 | Системы угроз | Все четыре: климат+тепло, болезни+травмы, рейды+оборона, психика+социум |
| 4 | Пространство | Полное: комнаты, распространение тепла, позиционный бой, pathfinding |
| 5 | Структура сессии | Длинный арк ~40–60 дней с эскалацией и финальной целью |
| 6 | Структура переработки | Подход C — архитектура систем + поставка фазами |

**Рекомендованные дефолты (одобрены):**

1. **Управление — приоритетное, без прямого микроконтроля колонистов** (игрок ставит приоритеты работ и зоны; ИИ сам строит пути и выбирает задачи).
2. **Здоровье — упрощённый список `conditions`** с выводом «способностей», а не полная иерархия частей тела.
3. **Инвентарь — глобальные пулы** для насыпного (дерево/еда/медикаменты/топливо) + **экипировка на колонисте** (одежда/оружие).
4. **День удлинить** (`TICKS_PER_DAY` 120 → ~240) под перемещение; финальная цифра — в балансе.

## 4. Архитектура: «движок систем» за тем же контрактом модуля

**Главный принцип:** жёсткое разделение **состояния** (плоские сериализуемые данные) и
**систем** (чистые функции, мутирующие состояние по `dt` + сид-RNG). Phaser-сцена —
тонкий хост: владеет `ColonyState`, прогоняет конвейер систем каждый тик, рендерит из
состояния, мостит команды/проекцию. React HUD — чистый view + эмиттер команд.

Контракт `GameModule` (`mount(container, ctx)` + `Hud`) **не меняется** — портал не трогаем.
`ctx` даёт `save / achievements / records / settings / events / exit` — как сейчас.

### 4.1. Модель состояния (`domain/types.ts`, всё сериализуемо)

```
ColonyState {
  version, seed, rngState, tick, day, phase, speed
  world: { w, h, tiles: Tile[] }
  colonists: Colonist[]
  buildings: Building[]
  resources: ресурсные пулы (food, wood, medicine, fuel, ...)
  env: { season, weather, tempCurve, ... }
  director: { tier, nextIncidentAt, ... }     // ИИ-рассказчик (Фаза 3+)
  arc: { goal, progress, ... }                  // финальная цель (Фаза 4)
  log: LogEntry[]
  flags: { gameOver, victory }
}

Tile { x, y, terrain, fertility, passable, buildingId?, roomId?, temp }

Colonist {
  id, name, backstory, traits: TraitId[]
  skills: Record<SkillId, { level, xp }>
  needs: { hunger, rest, warmth, recreation, mood }
  health: { conditions: Condition[], capacities: {...} }   // capacities выводятся
  agent: { task, target?, path: Pt[], pos: {x,y}, carrying? }
  equipment: { clothing?, weapon? }
  relations?: Record<colonistId, number>                    // Фаза 3
  alive
}

Building { id, type, footprint: Pt[], workSlots, storage?, heated?, hp }
Condition { id, kind: 'injury'|'illness', severity, progressing, infected? }
```

### 4.2. Конвейер систем (`systems/`)

Чистые функции, детерминированные через сид-`Rng`; выполняются по порядку каждый тик:

```
time → temperature(Ф1) → jobScheduler → pathfinding(A*) → work
     → needs → health(Ф2) → mood(Ф3) → combat(Ф3) → storyteller/events → winLose
```

- Колонисты **физически идут** к работе/еде/кровати.
- Нужды (голод/сон/тепло) перебивают низкоприоритетную работу.
- Навыки растут (xp) и влияют на выхлоп/скорость.
- Каждая система — отдельный тестируемый модуль (vitest).

### 4.3. Рендер (Phaser-сцена)

Тайлы (террейн + тинт комнат + оверлей температуры) · здания (footprint + глиф/спрайт) ·
колонисты как движущиеся по сетке агенты (реальные позиции вместо случайного блуждания) ·
враги в рейдах · клик по колонисту → инспектор. Камера pan/zoom. Проекция HUD троттлится
(как сейчас, ~150 мс).

### 4.4. React HUD

Расширяет существующие примитивы (`Button/Modal/ProgressBar/SectionTitle`):
ростер с детальным инспектором (нужды/навыки/черты/здоровье/экипировка) · вкладка
приоритетов работ · меню строительства (выбрал здание → клик по тайлу) · лента
инцидентов/тревог · сезон+температура · прогресс цели · скорость.

### 4.5. Сейвы, миграции, детерминизм

- Бампаем `payloadVersion` (2 → 3). Несовместимые старые сейвы уже отвергаются в `ColonyGameModule.mount` по версии — оставляем это поведение.
- Единственный источник случайности — сид-`Rng` с сохраняемым `rngState`. Один сид → идентичный прогон (основа воспроизводимых тестов).
- Ачивки/рекорды расширяем новыми (см. соответствующие фазы).

## 5. Фаза 0 — Фундамент (играбельная веха)

**Цель:** колонисты-личности реально живут и работают на сетке. Играется как мини-колония
даже без климата/боя.

### Содержание
- **Сетка и террейн со смыслом:** `grass/forest/rock/water` + `fertility` + `passable`. Фермы дают больше на плодородной земле; лес рубится у `forest`-тайлов и истощается; вода/камень непроходимы или дают камень.
- **Колонисты-личности:** имя + 1–3 черты из стартового пула (*Трудоголик, Хрупкий, Ленивый, Оптимист, Кровожадный, Неуклюжий*) · навыки (`farming, woodcutting, building, research, cooking, medicine, shooting, melee`) с уровнем+xp · нужды этой фазы: **голод и отдых** + позиция.
- **Работа = движение:** `jobScheduler` назначает свободного колониста на работу с наивысшим приоритетом → `pathfinding (A*)` строит путь → колонист идёт → `workSystem` применяет выработку по достижении, начисляет xp; навык влияет на скорость/выход. Нужда перебивает низкоприоритетную работу.
- **Приоритеты работ:** на колониста — матрица приоритетов (0–3) по типам работ + дефолтный авто-режим. UI компактный.
- **Строительство кликом:** выбрал здание → подсветка валидных тайлов → клик ставит (списывает ресурсы, занимает footprint). Стартовый набор: ферма, спальня (кровати), склад, лаборатория.
- **Рендер:** террейн, footprint зданий, колонисты-агенты идут по путям, клик → инспектор.
- **HUD:** ростер+инспектор, приоритеты работ, меню строительства, ресурс-бар (еда/дерево), скорость, лог. Заглушка победы/поражения (продержаться N дней).

### Критерии приёмки
- Колонист с приоритетом «фермер» доходит до фермы и поднимает запас еды; xp `farming` растёт.
- Голодный колонист бросает низкоприоритетную работу и идёт есть; при критическом отдыхе — спать.
- Здание ставится только на валидный тайл, ресурсы списываются, footprint занят и непроходим.
- Инспектор показывает корректные нужды/навыки/черты выбранного колониста.

### Тесты (TDD, vitest)
- `pathfinding`: путь по проходимым тайлам, обход преград, корректный «нет пути».
- `jobScheduler`: назначение по приоритету; перебивание нуждой; нет двойного захвата слота.
- `needs`: декей голода/отдыха; пороги переключения на еду/сон.
- `work`: выход и прирост xp как функция навыка/террейна.

## 6. Фаза 1 — Среда (играбельная веха)

**Цель:** климат — главный стратегический вызов; «подготовь колонию к зиме».

### Содержание
- **Комнаты:** замкнутые стенами+дверьми области получают `roomId` (flood-fill). Стена и дверь — новые типы построек.
- **Температура:** `temperatureSystem` распространяет тепло от обогревателей/костров по комнатам (итеративная релаксация по ячейкам); уличная температура — по сезону/погоде; дыра в стене = теплопотеря.
- **Нужда «тепло»:** из температуры тайла под колонистом + одежды. Холод → риск переохлаждения (стыкуется со здоровьем Фазы 2); жара → замедление работы.
- **Сезоны с последствиями:** зима замораживает уличные фермы и поднимает спрос на топливо; еда **портится** без закрытого/холодного хранения; **заготовка запасов на зиму — ядро стратегической петли**. Прогноз сезона в HUD.
- **Одежда:** базовые предметы на тепло, экипируются на колониста.
- **Постройки:** стена, дверь, обогреватель/костёр (топливо — дерево), склад с защитой от порчи.
- **HUD:** оверлей температуры, прогноз сезона, индикатор топлива/отопления.

### Критерии приёмки
- Замкнутая комната с обогревателем держит температуру выше уличной; пролом в стене её роняет.
- Зимой уличная ферма не даёт урожай; колонисты без тепла набирают риск переохлаждения.
- Еда вне защищённого склада портится со временем; в складе — сохраняется.

### Тесты (TDD, vitest)
- `temperatureSystem`: распространение тепла, удержание в замкнутой комнате, теплопотеря через проём.
- Переходы сезонов и их эффекты (заморозка ферм, спрос на топливо).
- Нужда тепла как функция темп+одежда; порча еды по условиям хранения.

## 7. Роадмап Фаз 2–4 (свои спеки)

**Фаза 2 — Здоровье.** Травмы (несчастные случаи, холод, бой) и болезни (инцидент-эпидемии)
как `conditions` с severity + прогрессией + риском инфекции. Медицинская работа: врач лечит
пациента в кровати, тратя медикаменты; навык `medicine` решает исход. Выздоровление ↔ смерть;
стойкие последствия снижают «способности». Переохлаждение/тепловой удар из Фазы 1
материализуются здесь.

**Фаза 3 — Угрозы и социум.** `storyteller` планирует рейды по тиру эскалации: враги
спавнятся у края карты, идут к колонии; **позиционный бой** (оружие, турели, стены как
преграды/укрытия); ранения уходят в систему здоровья; оборонительные постройки. `moodSystem`:
«мысли» от событий/нужд/среды → настроение → **психические срывы** (буйство, бегство, апатия)
с игровыми последствиями; отношения и соц-взаимодействия; нужда в досуге.

**Фаза 4 — Арк и контент.** Тиры эскалации по дню/богатству; финальная цель (напр. построить
и запитать **эвакуационный маяк** за несколько дней под кульминационный рейд/зиму, либо
пережить Гранд-зиму); баланс-проход; больше черт/событий/зданий/исследований; ачивки/рекорды;
экраны концовок.

## 8. Риски и снижение

- **Производительность** (A* + распространение тепла на 3×): малая сетка (~24–32²), кэш путей с пересчётом по требованию, инкрементальная релаксация температуры (не до сходимости за тик), троттлинг проекции HUD.
- **Сложность UI** (ростер/приоритеты/инспектор): переиспользуем существующие примитивы, компактные панели, инспектор в `Modal`.
- **Детерминизм** для сейвов: все системы чистые, единственный RNG — сид-`Rng` с сохраняемым `rngState`.
- **Объём Фазы 0** (модель + pathfinding + работа + рендер сразу): внутри спеки режем на под-шаги с TDD, каждый под-шаг даёт зелёные тесты.

## 9. Тест-стратегия

TDD по системам (vitest, как `tests/colony.test.ts` / `tests/scoring.test.ts`). Каждая
система — чистая функция → юнит-тест **до** реализации. Детерминизм-тест: один сид →
идентичный прогон фиксированного числа тиков. Ключевые наборы перечислены в Фазах 0–1.
