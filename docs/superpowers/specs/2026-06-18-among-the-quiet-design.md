# Among the Quiet — дизайн (соло-импостор, обман и психология)

- **Дата:** 2026-06-18
- **Статус:** дизайн утверждён, готов к плану реализации
- **Область:** новая третья игра портала. Жанр, которого нет: **обман и психология**.
  Ты — подсаженный саботажник в маленьком экипаже; читаешь людей и манипулируешь
  ими, выполняя тайное задание и не попадаясь.

---

## 0. Контекст и зачем

У портала две игры — Shadow Trace (детектив-дедукция) и Colony (выживание-сим).
Существующие концепты новых игр оказались однотипными (расследование/менеджмент).
Нужно свежее направление — выбрано **«обман и психология»** (скрытая инфа, блеф,
чтение оппонента, манипуляция). Классика жанра — мультиплеер; чтобы это работало
**соло в браузере**, психология экипажа **моделируется** — игрок против ботов с
характерами.

**Зафиксированные развилки (брейншторм):**
- **Форма:** структурная социалка на React, без физического движения. «Корабль» =
  расписания, локации-списком, разговоры, наблюдения, голосования. «Социальные шахматы».
- **Роль/цель:** импостор-саботажник; **скрытое задание** (украсть/сломать/подменить)
  за несколько дней **+ выжить** (не быть изгнанным). Без насилия, чистая инфильтрация.
- **Ритм:** день = тайм-слоты; в каждом выбираешь локацию + действие; NPC ходят по
  рутинам; совпали в локации → видят друг друга; периодически сбор + голосование.
- **Подозрение:** гибрид — читаешь по поведению/репликам, есть грубый само-индикатор
  «засвета»; точные числа скрыты.
- **Подход C** («гибрид: авторский каст/корабль + процедурное давление»): мир и
  персонажи спроектированы (честно, легко читать, драматично), социальная динамика
  эмерджентна и реиграбельна. Архитектура — «движок + контент + валидатор», как у
  Shadow Trace.

**Дизайн-столпы:**
1. **Честная сложность.** Сид-RNG, надёжные сигналы, восстановление из промаха — без
   «рандом убил».
2. **Размен прогресс↔засвет.** Работать на цель = делать подозрительное; надо выглядеть чистым.
3. **Движок не блефует игроку.** Сигналы NPC всегда правдивы; блефует только игрок.
4. **Подозрение из аномалий и co-presence**, не «по магии на игрока».
5. **Эмерджентная драма** — слухи по сети связей, козлы отпущения, каскады паранойи.

## Секция 1 — Опыт игрока: премиса и петля дня

**Сеттинг (дефолт, подвижный):** маленькая изолированная станция, тихий ночной экипаж
5–7 человек, отрезанный от мира. Ты — подсаженный саботажник; для всех ты свой.
Экипаж поначалу **доверяет** и не знает о чужом — подозрение рождается из **аномалий**,
которые плодит твоя работа. Гонка: завершить задание раньше, чем цепочка аномалий
укажет на тебя.

**Петля дня (тайм-слоты):**
1. **Утро.** Видишь: прогресс задания, грубый «засвет» (низко/средне/высоко), ночные слухи.
2. **Слоты (3–4 за день).** В каждом — **локация + действие**: *Слиться* (алиби, ↓засвет),
   *Двигать задание* (риск), *Разговор* (читать/направлять/прощупать), *Подсмотреть*
   (рутины/свидетели/материал для подставы). NPC одновременно по рутинам; совпали → видят.
3. **Развязка слота.** Движок считает co-presence, детект риска, скрытые сдвиги подозрения
   и выдаёт **читаемые сигналы** (взгляд, резкий ответ, реплика).
4. **Сбор и голосование** (конец дня или порог тревоги). Обвинения по подозрению+связям;
   ты **говоришь** (защита/перевод стрелок/обвинение/алиби); голос может кого-то изгнать.
   Изгнан невиновный → выжил, доверие перекраивается. Изгнан ты → конец.
5. Повтор по дням: задание готово → **победа**; изгнан/локдаун/время вышло → поражение.

**Ощущение:** тихий саспенс, социальный канат, каждое действие — размен прогресс↔засвет,
чтение лиц и слов, пряная (вымышленная) радость, когда за тебя «садится» невиновный.

**Экраны (React):** Станция (локации + кто там, трекер задания, засвет, день/слот);
Действие в слоте; Разговор (сигналы); Сбор/Голосование; Блокнот (что узнал о каждом).

## Секция 2 — Модель мира

```ts
interface Station {
  id; title; synopsis;
  difficulty: 'calm' | 'tense' | 'paranoid';
  days: number; slotsPerDay: number;
  locations: Location[];
  crew: NPC[];
  objectives: Objective[];      // ПУЛ; одно выбирается за забег
  cueLibrary: CueSet[];
  tuning: Tuning;
  startSuspicion: number;       // скрытый фон
}

interface Location {
  id; name;
  exposure: 'public' | 'semi' | 'private';   // насколько видны действия
  objectiveStepIds?: string[];
}

type Trait = 'nosy' | 'trusting' | 'gossip' | 'loner' | 'sharp' | 'anxious' | 'loyal';
interface WeightedLoc { locationId: string; weight: number }

interface NPC {
  id; name; role;
  traits: Trait[];
  routine: Record<number, WeightedLoc[]>;     // slotIndex → вероятные локации
  relationships: { npcId: string; bond: number }[];  // -100..100
  watchfulness: number;        // 0..1
  readStyle: string;           // ссылка на CueSet
}

interface Objective { id; title; steps: ObjectiveStep[] }
interface ObjectiveStep {
  id; label; locationId: string;
  baseRisk: number;            // насколько аномально/заметно
  requires?: string[];         // DAG предыдущих шагов
  leavesTrace?: { severity: number; discoverableBy?: 'sharp' | 'nosy' | 'any' };
}

interface Anomaly {
  id; fromStepId?; locationId; slot; day;
  severity: number; discoveredBy?: string;
}
```

**Принципы:** черты модулируют поведение (`nosy`↑watchfulness, `gossip` разносит слухи,
`trusting` медленно подозревает, `sharp` связывает аномалии, `loner` мало свидетелей —
лёгкая подстава, `anxious` колеблется на голосе, `loyal` голосует за друзей). Рисковый
шаг в `public` почти наверняка увидят; в `private` безопаснее, но «зачем полез в глухой
отсек» — поведенческая аномалия, если засекли по пути. **Подозрение цепляется к аномалии
и co-presence, а не к игроку напрямую** — поэтому алиби и подстава = «перенос свидетельства».

## Секция 3 — Движок подозрения, слухов, голосования, «чтение»

Чистый детерминированный движок (сид-RNG `core/utils/rng.ts`).

**1. Разрешение слота.** Расстановка по рутинам (+ дрожание по сиду) → co-presence →
детект риска: `P(заметили) = f(baseRisk, exposure, watchfulness свидетелей, сид-roll)` →
наблюдения у NPC, возможно новая `Anomaly`. Сид → честно и воспроизводимо.

**2. Подозрение.** У каждого NPC приватная карта `suspicion[кто]`. Растёт: прямое
свидетельство; обнаружение следа + ты был co-present; противоречие алиби; полученные
слухи (вес = доверие к источнику). Падает: подтверждённое алиби; доверие; время без инцидентов.

**3. Слухи.** NPC делятся топ-подозрениями с положительно связанными; получатель обновляет
с весом по доверию к источнику и своим чертам (`trusting` гасит, `sharp` усиливает,
`gossip` разносит). Ограниченные итерации — без самораскрутки.

**4. Сбор и голосование.** Обвинения = топ-подозрения; «давка» (`anxious` примыкает к
большинству, `loyal` не сдаёт друзей). **Реплика игрока** сдвигает распределение этого
голосования. Голос: каждый за самого подозреваемого (с bond); ничья → никого. Изгнан
игрок → поражение. Изгнан невиновный → выбывает (рутины/связи меняются); даёт ложное
«закрыто» либо, если позже `sharp` найдёт след, разжигает паранойю.

**5. Чтение (гибрид).** Чисел нет. Состояние NPC проступает через **авторские сигналы**
по полосе подозрения `trust → wary → cold → accusing`: тон реплик, тейлы (взгляд,
замолкает, садится дальше, чьё имя называет). Плюс грубый **само-индикатор засвета**
(низко/средне/высоко) — рельс, чтобы гибрид не был глухим.

**Честность:** движок **не блефует игроку** — сигнал «холодно» = подозрение реально
поднято. Блефует только игрок. Чтение всегда вознаграждается.

**Чистые функции:**
```ts
resolveSlot(station, state, playerAction, seed) → state
gossipPhase(station, state, seed) → state
readCues(station, state, npcId) → Cue[]
selfExposure(state) → 'low' | 'med' | 'high'
runMeeting(station, state, playerSpeech, seed) → { ejectedId?: string; state }
```

## Секция 4 — Инструментарий обмана игрока

Каждый слот = локация + действие.

```ts
type PlayerAction =
  | { kind: 'blend'; locationId: string }
  | { kind: 'objective'; stepId: string }
  | { kind: 'talk'; npcId: string; tack: 'reassure' | 'seed' | 'fish'; targetId?: string }
  | { kind: 'observe'; locationId: string; mode: 'routine' | 'witness' | 'plant'; targetId?: string };

type Speech =
  | { kind: 'alibi'; slotRef: { day: number; slot: number } }
  | { kind: 'accuse'; targetId: string }
  | { kind: 'sow_doubt' }
  | { kind: 'quiet' };
```

- **Слиться** — проверяемое алиби, ↓подозрение со-присутствующих, ↓засвет. Сильнейшая
  защита, но слот не двигает задание (давление времени).
- **Двигать задание** — `ObjectiveStep` (если `requires` готовы). Если заметили → `Anomaly`.
  `private` ↓шанс свидетеля, но «полез в отсек» = поведенческая аномалия при засвете.
- **Разговор** — *reassure* (рапорт ↓его подозрение), *seed* (вкинуть подозрение на
  `targetId`), *fish* (узнать что видел/рутину). Разговор = алиби. `sharp` ловит топорный
  посев → обратка.
- **Подсмотреть** — *routine* (выучить рутину), *witness* (кто что видел), *plant*
  (подбросить материал на невиновного). В `private` риск «застукали за подсматриванием».

**Подстава** — накопленный рычаг (посевы + `plant` + естественная co-presence жертвы у
аномалии), движок направляет поток подозрения на цель. `loner` — легчайшая мишень.

**Реплики на сборе:** *Алиби* (реальный слот-блеф со свидетелями ↓подозрение; фальшивое
опровергнут → обратка); *Обвинить* (сдвиг давки, сильнее при накопленном рычаге; без
опоры → твоё подозрение ↑); *Посеять сомнение* (стоп-голос, время, но тревога остаётся);
*Молчать* (безопасно, но не влияешь). Эффект модулируется чертами (`anxious` колеблется,
`sharp` сопротивляется, `loyal` якорит).

**Самообладание (`composure`)** — личный тейл. Ложь/риск тратят; блеф/отдых восстанавливают.
Низкое → твоя нервозность заметнее → watchfulness экипажа к тебе ↑. Страховка от спама риска.

## Секция 5 — Авторинг станции + процедурная вариация

Станция — один JSON; движок крутит с сидом.

**Библиотека сигналов** (то, через что проступает подозрение):
```ts
interface CueSet {
  readStyle: string;           // ссылается NPC.readStyle
  bands: { trust: Cue[]; wary: Cue[]; cold: Cue[]; accusing: Cue[] };
}
interface Cue {
  channel: 'dialogue' | 'tell' | 'meeting';
  text: string;                // «садится дальше», «—Ты дёрганый…», смотрит на {target}
  traitGate?: Trait;
}
interface Tuning {
  suspicionGain: number; gossipStrength: number; detectionMult: number;
  paranoiaEscalation: number; composureCost: number; composureRegen: number;
}
```

**Что варьируется за забег (по сиду):** выбор одного `Objective` из пула; дрожание рутин;
шум на стартовых `bond`/`startSuspicion`; смещение «естественного козла отпущения»
(эмерджентно). **Каст и характеры фиксированы** (авторская, честная часть); процедурна
только динамика.

**Валидатор `validateStation`** (честность и проходимость, как «нет тупиков» у Shadow Trace):
- структура: все `locationId` рутин, `npcId` связей, локации/`objectiveStepId` шагов,
  ссылки `readStyle` резолвятся; нет дублей id;
- покрытие сигналов: у каждого `readStyle` ≥1 `Cue` на каждую полосу (trust/wary/cold/accusing);
- решаемость: для каждого `Objective` в пуле существует расписание в пределах `days×slots`,
  завершающее все шаги с ненулевым шансом выжить (DAG `requires` ацикличен и проходим; у
  каждого рискового шага есть `private/semi`-окно, где детект не гарантирован);
- баланс: есть правдоподобный козёл отпущения (`loner`/слабый bond) — иначе предупреждение;
- границы: `days≥2`, `slotsPerDay≥3`, `crew≥4`.
Падение валидатора = станция не собирается.

## Секция 6 — Победа/поражение, сложность, длина

**Победа:** все шаги задания **+** дожил до эвакуации. **Ранг:** `Призрак` (ни одна
аномалия не обнаружена) → `Чисто` → … → `Грязно` (цель достигнута, но за тебя «сел»
невиновный / паранойя). Мораль — выбор, не штраф.

**Поражение:** **изгнан** на голосовании; **локдаун** (подозрение к тебе на максимуме
раньше, чем готово задание); **время вышло**.

**Без смерти за одну ошибку:** один засвет не конец; восстановление алиби/подставой/временем.
Изгнание требует устойчивого подозрения + удачно легшего голоса.

**Сложность (`Tuning`-пресеты `calm/tense/paranoid`):** watchfulness, скорость слухов,
детект, темп паранойи, экономика `composure`, число шагов, бюджет дней. **`nightmare`:**
сигналы скупее + снят рельс засвета.

**Длина:** станция ≈ 15–25 мин (~4 дня × 4 слота). Реиграбельность: сид-вариация,
разные пути (осторожный vs агрессивный), эмерджентная драма, тиры, несколько станций.
**Мета (карьера-рогалик) отложена по YAGNI**; MVP = одна станция + ранг + достижения.

## Секция 7 — Архитектура, интеграция, поэтапный план, тесты

**Архитектура** (паттерн «движок + контент + валидатор», pure-React-игра):
- `src/games/among-the-quiet/` — новый модуль.
- `engine/`: типы; состояние; сид-RNG (`core/utils/rng.ts`); `resolveSlot` / suspicion /
  `gossipPhase` / `selfExposure` / `runMeeting` / `readCues`; `validateStation`. Чистое, тестируемо.
- `public/data/stations/*.json` — авторские станции.
- `ui/`: 5 экранов (Станция, Слот, Разговор, Сбор/Голосование, Блокнот) — тонкие.
- Контракт `GameModule` (`mount` no-op, `Hud`); регистрация в `GameRegistry`; ленивый чанк.

**Состояние:**
```ts
interface GameState {
  station: Station; day: number; slot: number;
  phase: 'plan' | 'resolve' | 'meeting' | 'result';
  crew: CrewMemberState[];          // suspicion-карты, наблюдения, alive
  player: { composure: number; objectiveProgress: string[]; notebook: NotebookEntry[]; alibis: AlibiRecord[] };
  anomalies: Anomaly[];
  objectiveId: string;              // выбранное из пула
  seed: number; rngCursor: number;
}

interface CrewMemberState {
  npcId: string;
  alive: boolean;                   // не изгнан
  suspicion: Record<string, number>;   // к каждому (вкл. 'player'); СКРЫТО от игрока
  observations: Observation[];      // что лично видел
}
interface Observation { day; slot; locationId; subjectId; kind: 'present' | 'odd_action' | 'anomaly_found' }
```
`NotebookEntry` (что игрок узнал о каждом NPC — рутина, связи, чтение) и `AlibiRecord`
(слот+локация+свидетели для защиты на сборе) — производные структуры, финальная форма
фиксируется в плане Этапа 0. Переходы — чистые функции (как у case-движка Shadow Trace).

**Интеграция с порталом:**
- `GameId` += `'among-the-quiet'`; новая `GameTheme` `'quiet'` (холодная палитра, `data-theme`).
- `GameDefinition` (tags: `['social','deception','psychology']`).
- Достижения: ранг `Призрак`, «без единого blend», «подставил невиновного», «прошёл `nightmare`».
- Настройки: сложность; `reduced-motion`. Сейв: забег = автосейв каждый слот (`ctx.save`);
  ранги/счётчики (`ctx.records`).

**Поэтапный план (вертикальный срез вперёд):**
- **Этап 0 — Фундамент движка (без UI):** типы, состояние, сид-RNG, `resolveSlot`,
  обновление подозрения, `gossipPhase`, `selfExposure`, `runMeeting`, `readCues`,
  `validateStation`; тест-станция-фикстура. Всё чистое + юнит-тесты. Основной объём.
- **Этап 1 — Срез UI:** 5 экранов на движке, одна станция end-to-end, одна сложность.
  Проходимый забег: слоты → развязка → чтение сигналов → сбор/голос → победа/поражение + ранг.
- **Этап 2 — Глубина:** подстава/`plant`, тейлы `composure`, пул заданий, ширина библиотеки
  сигналов, локдаун, `nightmare`.
- **Этап 3 — Контент и мета:** 2–3 станции, достижения, опц. карьера-рогалик.

**Тесты** (vitest, детерминированно через сид):
- юнит: детект `resolveSlot`; сходимость/границы `gossipPhase`; изгнание `runMeeting`;
  полоса `readCues`; агрегат `selfExposure`; `validateStation` (решаемость, покрытие
  сигналов, висячие ссылки, дубли id);
- свойство: каждая станция проходит `validateStation`;
- регресс: один сид + те же входы → тот же исход.
- UI — запуском приложения (jsdom/RTL не настроены).

**Переиспользование:** сид-RNG, `GameModule`/`PortalBridge`/`SaveManager`/`AchievementManager`,
паттерн «движок+контент+валидатор», ленивый чанк, Tailwind-токены + новая тема.

## Открытые вопросы / решения по умолчанию

- **Объём Этапа 1.** По умолчанию: 1 станция, 5 NPC, 5–6 локаций, 1 задание из 3 шагов,
  4 дня × 4 слота, сложность `tense`. Корректируется при написании плана.
- **Тема/сеттинг.** Дефолт — изолированная станция/«ночная смена»; финальный антураж
