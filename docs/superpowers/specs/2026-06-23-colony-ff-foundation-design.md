# Colony → Farthest-Frontier-гибрид · Фундамент (большой мир и движок): дизайн

- **Дата:** 2026-06-23
- **Игра:** `colony` (в коде — «Colony Survival»)
- **Статус:** дизайн одобрен по секциям в брейншторме; ожидает ревью спеки перед планом реализации
- **Подход:** чистая архитектура систем + поставка играбельными вехами (как в предыдущих фазах)
- **Предшествует:** Фазы 0 (Foundation) и 1 (Environment) — завершены и зелёные. Этот спек **пере-фундаментирует** движок под масштаб и новое направление.

---

## 1. Контекст и видение

Игра уже переработана в RimWorld-lite (карта 28², колонисты-личности, A*-пути, климат/зима — Фазы 0–1). Новое направление по запросу игрока — **сдвинуть игру в сторону Farthest Frontier**: town-scale градострой-выживание на **огромной карте 256²** с населением **200+**, улучшенной графикой (**top-down 2.5D, процедурные текстуры**) и большим объёмом геймплея (биомы, залежи, добыча, фермерство вглубь, производственные цепочки, торговля, тиры города, оборона).

Это слишком велико для одного спека, поэтому работа декомпозирована на под-проекты (см. §15). **Данный спек покрывает только Фундамент** — техническую базу, без которой остальное нереализуемо: большой процедурный мир, новый рендер, поиск пути и симуляция на масштабе. Существующие системы (нужды/работа/стройка/климат) переезжают на новый движок без потери поведения.

### Направление (лог брейншторминга)

| # | Вопрос | Решение |
|---|--------|---------|
| 1 | Жанровый сдвиг | **Гибрид как Farthest Frontier**: town-scale; жители именованные с домами/здоровьем/нуждами, но без микро-личностей и психических срывов |
| 2 | Масштаб карты/населения | **256×256 (~65k тайлов), население 200+** |
| 3 | Арт-направление | **Top-down 2.5D + процедурные текстуры** (без сорсинга ассетов; изометрика отклонена по риску FPS/ассетов) |
| 4 | Приоритет контент-столпов | Первый — **«Земля, добыча, фермы»** (порядок столпов в §15) |
| 5 | Хранилище карты | **SoA (типизированные массивы) + регенерация из сида** (террейн не сериализуется) |
| 6 | Композиция мира | Умеренный FF-ландшафт (вода/реки, болото, луга, трава, лес, скалы, горы), залежи по биомам — макет одобрен |

## 2. Границы Фундамента

**Входит:**
- Мир 256² генерируется процедурно: биомы, водоёмы/реки, леса, **видимые залежи** (камень/глина/руда/золото/ягоды/рыба) как данные на карте.
- Новый рендер top-down 2.5D + процедурные текстуры; камера pan/zoom; миникарта.
- Поиск пути и симуляция масштабируются до 200+ агентов без просадок (иерархический pathfinding, спатиал-индексы, тайм-слайс).
- Существующие системы (нужды/работа/стройка/комнаты/температура/сезоны/порча) переезжают на SoA-движок; поведение сохраняется; детерминизм и сейвы работают (с бампом версии).
- Стартовая площадка выбирается детерминированно; стартовые колонисты + небольшой запас.

**НЕ входит (уходит в столпы, см. §15):**
- *Геймплей эксплуатации земли*: зоны рубки/добычи/собирательства, designations, фермерство вглубь (плодородие/расчистка/севооборот), переработка. Фундамент лишь раскладывает залежи/биомы и «крючки».
- Охота/рыбалка/болезни/вода, производственные цепочки/торговля/тиры, оборона/рейды.
- Рост населения (иммиграция/рождения) — стартовое население фиксировано; рост придёт со столпом «Производство/город».

**Веха:** на выходе Фундамента игрок исследует гигантскую живую карту в новом визуале; его колонисты ходят и работают как в Фазах 0–1, но на новом масштабе. Самостоятельная играбельная веха.

## 3. Модель состояния (`domain/types.ts`)

**Принцип:** карта — чистая функция от `seed`. В сейв пишутся только сид + мутации; сетка регенерируется на загрузке.

### 3.1. Сетка — структура массивов (SoA)

```
map: {
  w: number; h: number;
  terrain:    Uint8Array;   // biome-энум на тайл (см. 3.2)
  elevation:  Float32Array; // 0..1 — 2.5D-тень, вода, склоны
  fertility:  Float32Array; // 0..1 — урожайность (для фермерства-столпа)
  passable:   Uint8Array;   // 0/1 — производное (реконструируется из биома+построек)
  roomId:     Uint16Array;  // считается системой комнат (не сохраняется)
  temp:       Float32Array; // считается системой температуры (не сохраняется)
  buildingAt: Int32Array;   // индекс здания в state.buildings или -1
  nodes: Map<number, ResourceNode>; // РАЗРЕЖЕННО: tileIndex → залежь (мутируемо/сохраняемо)
}
// idx = y*w + x; доступ через аксессоры grid.ts
```

```
ResourceNode { kind: NodeKind; amount: number; max: number }
NodeKind = 'wood' | 'stone' | 'clay' | 'iron' | 'gold' | 'berries' | 'fish'
```

Биом-энум (`Uint8`): `0 water · 1 marsh · 2 meadow · 3 grass · 4 forest · 5 rock · 6 mountain`. (Снежные пики из макета — визуальный вариант `mountain` по высоте, не отдельный биом.)

### 3.2. Прочее состояние

- `Colonist`, `Building`, `Room`, `env`, `resources`, `stock` — как в Фазах 0–1, поля сохраняются. `pos`/`path` остаются (координаты тайлов float).
- `ColonyState.map` меняет форму с `{ w,h, tiles: Tile[] }` на SoA (3.1). Интерфейс `Tile` остаётся как **транзитный view** для холодных путей (инспектор/отладка), собираемый из SoA по запросу — но горячие циклы используют индексные аксессоры, не аллоцируют `Tile`.
- Производные структуры (кластеры/порталы/бакеты целей, см. §5–6) живут вне сериализуемого `ColonyState` (строятся на загрузке).

### 3.3. Сейвы и детерминизм

- `payloadVersion` бампается; несовместимые старые сейвы отвергаются в `ColonyGameModule.mount` (поведение уже есть).
- **Граница сериализации (обязательна — `Map`/типизированные массивы не сериализуются в JSON напрямую):** вводим `toSave(state)`/`fromSave(payload)`. `toSave` исключает SoA-массивы и производные структуры (кластеры/бакеты), конвертирует `map.nodes` (`Map`) в массив `[tileIndex, node]`-записей. `fromSave` зовёт `regenerateWorld(seed)` и накатывает мутации.
- Сейв-пейлоад = `seed + rngState + colonists + buildings + nodes-записи + resources/stock/env/flags/log`. SoA-сетка **не** сериализуется.
- Загрузка: `regenerateWorld(seed)` → накат `nodes`-записей и `buildingAt` из `buildings` → построение производных → пометка грязных кластеров.
- Единственный источник случайности — сид-`Rng` с сохраняемым `rngState`. Тайм-слайс назначений и обход индексов — в фиксированном порядке. Один сид → идентичный прогон (основа тестов).

## 4. Генерация мира (`domain/worldgen.ts`, `core/utils/noise.ts`)

Детерминированная, чистая `regenerateWorld(seed) → map`. Новый `noise.ts` — сид-сеяный value-noise (fbm, многооктавный), покрыт тестами.

Слои:
1. **Высота** = fbm(seedE). Низины (< порога) → вода.
2. **Влажность** = fbm(seedM, со смещением).
3. **Реки:** из нескольких высоких точек спуск по градиенту высоты к воде → водные артерии + плодородно-глинистые берега.
4. **Биомы:** по таблице порогов (высота × влажность): вода/болото/луга/трава/лес/скалы/горы. (Параметры порогов — в `data/balance.ts`.)
5. **Залежи (`nodes`):** разброс по правилам биома/высоты — камень/железо/золото в скалах/горах, глина у воды/болот, дерево в лесу, ягоды в лугах/у леса, рыба в воде. Детерминированный отдельный RNG-поток.
6. **Плодородие:** из биома + влажности (луга высоко, скалы низко).
7. **Проходимость:** производная (вода/горы/каменные выходы непроходимы; узлы могут блокировать до добычи — флаг на узле/биоме).
8. **Стартовая площадка:** детерминированный поиск ровного плодородного пятна рядом с водой + лесом + камнем (ближе к центру); туда — стартовые колонисты + запас; камера центрируется.

## 5. Архитектура: рендер-пайплайн (`scenes/render/`)

`WorldScene` дробится на модули (изоляция ответственностей):
- **`ChunkRenderer`** — карта режется на чанки 32×32; статический террейн чанка запекается один раз в `RenderTexture` из процедурных текстур биомов (набор затайленных шумовых текстур + затенение по `elevation`). На сцену добавляются только видимые чанки (камера-куллинг); при pan — соседние.
- **`SpriteLayer`** — здания/деревья/залежи/колонисты как пуллингованные спрайты в одном слое с `depth = y` (**Y-sort**) для 2.5D-перекрытия; тени отдельным смещённым элементом. LOD: на дальнем зуме — упрощённый рендер (точки/без теней).
- **`Minimap`** — запечённая уменьшенная биом-текстура + точки колонистов/зданий + прямоугольник вьюпорта; клик/драг = перенос камеры.
- **`CameraController`** — pan (drag/края/WASD) + zoom (колесо), ограничение по миру.
- **Оверлеи** (температура; зоны/designations — в столпах) — отдельный троттлящийся слой поверх, чанково/куллингом.

Рендер строго из состояния; ввод (клик-выбор/стройка) мостится в команды как сейчас.

## 6. Архитектура: поиск пути и спатиал-индексы

- **`pathfinding.ts`:** A* на бинарной куче (замена линейного open-list).
- **`pathHierarchy.ts` (новый):** кластеры 16×16; на границах — порталы (проходимые проёмы) и предпосчитанные внутрикластерные расстояния портал↔портал. Длинный путь: A* по графу порталов → локальный A* «старт→портал» и «портал→цель». Корректность: путь валиден и не длиннее K·оптимума (K=`PATH_LEN_BOUND_K`), сверяется с BFS на малых картах (тест). HPA* околооптимален — равенство с полным A* не требуется.
- **Кэш путей + ленивая инвалидация:** при изменении проходимости (стена/постройка/добыча) пересчитывается только грязный кластер и его порталы; кэш сбрасывается точечно.
- **`spatialIndex.ts` (новый):** здания индексируются по `jobType`; ресурс-узлы — в бакетах кластера по `NodeKind`. «Ближайшая цель» = расширяющийся поиск по кольцам бакетов от позиции (локально, не по всей карте). Заменяет `for t of all tiles` в `jobScheduler`.
- Flow-field откладывается на столп «Оборона».

## 7. Масштаб симуляции (200+ агентов)

- `stepAgents` (движение по пути) — дёшев, каждый тик.
- **Тайм-слайс назначений:** поиск цели + построение пути для idle — бюджет N на тик; остальные idle ждут следующего. Порядок обхода фиксирован (детерминизм). Срезает пики без видимой задержки.
- `recomputeRooms` — по bbox изменения, не флуд по 65k.
- HUD-проекция остаётся троттленной (~150 мс); ростер на 200 — сводка населения + инспектор выбранного (полноценный список — UX столпов).

## 8. Конвейер тика и загрузка

Порядок систем прежний (`tick.ts`): `outdoorTemp → rooms(lazy) → temperature → needs → jobScheduler → stepAgents → work → onNewDay`. Изменения:
- На старте/загрузке: `regenerateWorld(seed)` → построение кластеров/порталов/бакетов (производные, не сохраняются) → накат мутаций → грязные кластеры.
- `jobScheduler` использует `spatialIndex` + бюджет; стройка/добыча помечают грязные кластеры.
- Совместимость сейвов: `payloadVersion` бамп; старые отвергаются.

## 9. Критерии приёмки

- Генерируется связная карта 256² с биомами/реками/залежами; стартовая площадка проходима, колонисты не заспавнены запертыми.
- Камера свободно ходит по миру; миникарта отражает мир и вьюпорт; клик по миникарте переносит камеру.
- Колонист доходит до удалённой цели через иерархический путь; путь валиден и околооптимален (≤ K·оптимума); при постройке стены путь пере-прокладывается.
- 200+ колонистов симулируются и рендерятся без исключений и без явных просадок (детерминизм-смоук зелёный); тайм-слайс не ломает поведение.
- Существующее поведение Фаз 0–1 (работа/еда/сон/комнаты/температура/зима/порча) воспроизводится на новом движке.
- Сейв/загрузка: мир регенерируется из сида идентично; истощение узлов и постройки сохраняются.

## 10. Тест-стратегия (TDD, vitest)

- `noise`/`worldgen`: сид → идентичные биомы/старт; распределение залежей в допусках; стартовая площадка связна/проходима.
- `pathfinding`/`pathHierarchy`: иерархический путь валиден и не длиннее K·оптимума (vs BFS) на малых картах; обход преград; «нет пути»; инвалидация по грязному кластеру.
- `spatialIndex`: «ближайшая цель» == брутфорс на малых картах; обновление при изменениях.
- SoA-аксессоры (`grid.ts`): согласованность terrain/fertility/passable; регенерация == исходник; персист истощения узлов через сейв/загрузку.
- Детерминизм-смоук: 256² + N агентов прогоняются X тиков без исключений; один сид → идентичный прогон.
- Существующие наборы (`needs/work/temperature/rooms/season/cold/spoilage`) адаптируются к SoA — остаются зелёными.

## 12. Производительность: целевые ориентиры

- Цель: плавный рендер видимой области (чанки+куллинг) и симуляция 200+ агентов без исключений; точные FPS-числа — на дев-бенч внутри реализации (не блокируют архитектуру).
- Главные рычаги: чанковый запечённый террейн, иерархический путь + кэш, спатиал-индекс целей, тайм-слайс назначений, троттлинг проекции, LOD на дальнем зуме.

## 13. Роадмап столпов (контекст декомпозиции)

После Фундамента — отдельные под-проекты (каждый: свой брейншторм-спек → план → реализация):
1. **Земля, добыча, фермы** *(первый по приоритету игрока)* — биомы/залежи в дело: зоны рубки/добычи/собирательства, designations, фермерство вглубь (плодородие/расчистка/севооборот).
2. **Еда и здоровье** — охота/рыбалка/собирательство, разнообразие рациона→здоровье, хранение/порча (погреб/коптильня), болезни (дизентерия/холера/крысы), чистая вода/колодцы, лечение.
3. **Производство, экономика, тиры города** — цепочки сырьё→переработка→товары, склады/логистика, торговец/золото, тиры поселения (лагерь→деревня→город) с разблокировкой зданий, рост населения.
4. **Оборона и налётчики** — стены/ворота/башни, ополчение/солдаты, волны рейдов с эскалацией (здесь подключается flow-field).

Фундамент закладывает «крючки» под (1): залежи как данные, биомы/плодородие, грязные кластеры под designations.

## 14. Риски и снижение

- **Производительность на 256²/200+** — главный риск: смягчается чанковым запечённым рендером, иерархическим путём+кэшем, спатиал-индексами, тайм-слайсом, LOD; дев-бенч внутри реализации.
- **Объём рефактора SoA** (все системы читают `tiles[i].field`): вводим аксессоры `grid.ts` как единый шов; миграция систем — механическая, под зелёными адаптированными тестами.
- **Корректность иерархического пути:** сверяется с полным A* на малых картах юнит-тестами.
- **Корректность генерации (запертый старт, связность):** покрыто тестами worldgen.
- **Совместимость сейвов:** `payloadVersion` бамп; старые отвергаются (как уже устроено); сетка регенерируется из сида.
- **Дробление `WorldScene`:** разнесение по `render/*` снижает риск «слишком большого файла» и упрощает реализацию субагентами.

- Текстурный конвейер: стартуем с процедурных шумовых текстур; рисованные/ассетные тайлы — опционально позже (вариант C из арт-выбора).
- Точные FPS-цели и целевое «железо» — фиксируются дев-бенчем в реализации.
