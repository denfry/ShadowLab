## 1. Цель и место в арке

**Цель Фазы 1:** климат становится главным стратегическим вызовом — «подготовь колонию к зиме».
Игрок строит замкнутые отапливаемые комнаты, шьёт тёплую одежду, запасает еду до того,
как зима заморозит уличные фермы и начнёт убивать незащищённых колонистов.

Опирается на движок систем Фазы 0 (плоское `ColonyState` + чистые системы в конвейере тика,
сид-`Rng`, Phaser-рендер + React HUD). `payloadVersion` 3 → 4 (старые сейвы отвергаются).

## 2. Принятые решения (брейншторминг Фазы 1)

| # | Вопрос | Решение |
|---|--------|---------|
| 1 | Модель температуры | **По-комнатно, равномерно**: у замкнутой комнаты одна температура = f(обогреватели, размер, улица). Дешёво, предсказуемо. По-ячеечная диффузия — возможна позже. |
| 2 | Порча еды | **Глобальная свежесть**: пул еды теряет долю в день вне холода/склада; склад и зимний холод резко замедляют порчу. |
| 3 | Одежда | **Одна тёплая вещь + пошив**: один слот «тёплая одежда» на колониста, шьётся на верстаке (новая работа `tailor`); даёт +тепло. |
| 4 | Эффект холода | **Нужда + замедление + урон при замерзании**: холод поднимает нужду `cold`, замедляет работу; при сильном/долгом морозе — простой урон по `health` (как голод). Полная система conditions/переохлаждения — Фаза 2. |

**Доп. решения (фиксирую, опираясь на простоту):**
- **Топливо для обогрева — это `wood`** (двойное назначение: стройка + тепло), без нового ресурса. Создаёт зимнюю тензию вокруг дерева.
- **Зима замораживает фермы на холодных тайлах**: ферма производит, только если температура её тайла > 0 °C. Ферма в тёплой комнате (теплица) продолжает работать — связывает комнаты+тепло с едой.
- **Согрев пассивный**: колонисты не ищут тепло специальным поведением (как еду/сон) в Фазе 1 — тепло набирается, пока колонист в тёплом тайле (работа в отапливаемой комнате, сон в тёплой спальне). «Иди греться» — кандидат на позже.

## 3. Изменения модели состояния (`domain/types.ts`)

Добавления (Фаза 0 поля сохраняются):

```
// Сезон/погода/уличная температура
env: {
  season: 'spring' | 'summer' | 'autumn' | 'winter';
  dayInSeason: number;
  outdoorTemp: number;        // текущая уличная °C (день/ночь/погода)
  weather: 'clear' | 'snow' | 'cold_snap';
}

// Tile: добавить
roomId: number;               // 0 = улица; >0 = id комнаты
temp: number;                 // °C текущая температура тайла

// Room (новая сущность; перестраивается при изменении стен/дверей)
rooms: Room[]                 // в ColonyState
Room { id: number; tiles: number[]; temp: number; area: number; }

// Colonist.needs: добавить
cold: number;                 // 0 (комфорт) .. 100 (замерзает), выше = хуже

// Colonist: добавить
clothed: boolean;             // надета ли тёплая одежда

// Ресурсы: добавить пул готовой одежды (не °C, а штуки)
resources.clothing? — НЕТ: одежду держим как отдельный счётчик stock.clothing: number
// (не ResourceId, т.к. без capacity/бара ресурсов). Поле: state.stock = { clothing: number }

// Новые типы зданий
BuildingType += 'wall' | 'door' | 'heater' | 'tailor'
// JobType += 'tailor'
```

Заметки по полям:
- `wall` — непроходимо (passable=false на его тайле). `door` — проходимо, но барьер для определения комнат и удержания тепла.
- `heater` — потребляет `wood` пока активен; греет свою комнату. `tailor` — верстак, на нём работа `tailor` шьёт одежду из `wood`.
- `stock.clothing` — глобальный счётчик готовой тёплой одежды; колонист «надевает» (clothed=true), расходуя 1, когда мёрзнет и одежда есть.

## 4. Новые/обновлённые системы

Порядок в конвейере тика (вставки помечены **NEW**/**upd**):

```
time → season(upd onNewDay) → room(NEW, ленивo) → temperature(NEW)
     → jobScheduler(upd: +tailor) → pathfinding → work(upd: +tailor, +зимняя заморозка ферм, +топливо обогрева)
     → needs(upd: +cold) → winLose
```

### 4.1. `seasonSystem` (в `onNewDay` или отдельный модуль)
- Каждый день: `dayInSeason++`; при достижении `SEASON_LENGTH` — следующий сезон, `dayInSeason=0`.
- Базовая уличная температура по сезону: весна +12, лето +26, осень +8, зима −12 °C.
- Суточная/погодная вариация: ночь холоднее на ~6°; `cold_snap`/`snow` снижают ещё. Детерминировано через сид-`Rng`.
- `outdoorTemp` пересчитывается каждый тик из сезона+фазы(день/ночь)+погоды (дёшево).

### 4.2. `roomSystem` (ленивый — пересчёт только при изменении набора стен/дверей)
- Flood-fill от границ карты через тайлы, НЕ являющиеся `wall`/`door` → помечает «уличные» (roomId=0).
- Оставшиеся непустые внутренние тайлы группируются в комнаты (компоненты связности через не-`wall` тайлы; `door` — барьер для нумерации комнат). Каждой комнате — id, список тайлов, площадь.
- Триггер пересчёта: изменился счётчик/состав `wall`+`door` (как `buildSig` в сцене Фазы 0).

### 4.3. `temperatureSystem` (по-комнатно, каждый тик)
- Для каждой комнаты: `target = outdoorTemp + heatPower`, где `heatPower = Σ(активные обогреватели в комнате) * HEATER_OUTPUT / max(1, area / AREA_NORM)` (большую комнату греть труднее).
- `room.temp` релаксирует к target: `room.temp += (target - room.temp) * TEMP_LERP`.
- Тайлы комнаты получают `room.temp`; уличные тайлы — `outdoorTemp`.
- **Топливо:** активный обогреватель сжигает `HEATER_FUEL_PER_TICK` дерева; если дерева нет — неактивен (не греет). Списание из `resources.wood`.

### 4.4. `needs` (обновление — добавить `cold`)
- Эффективная температура колониста = `tileTemp(под ним) + (clothed ? CLOTHING_WARMTH : 0)`.
- Если `effTemp < COMFORT_MIN` (напр. 16°): `cold` растёт пропорц. недостатку тепла; иначе `cold` падает.
- Эффекты: `cold >= COLD_SLOW_THRESHOLD` → множитель скорости работы < 1 (используется в `work`). `effTemp <= FREEZING` (напр. ≤ −2°) или `cold` максимально → урон по `health` (как голод). 
- Авто-одевание: если `cold >= CLOTHE_THRESHOLD`, `!clothed`, `stock.clothing > 0` → `clothed=true`, `stock.clothing -= 1`.

### 4.5. `work` (обновления)
- **Зимняя заморозка ферм:** ферма производит еду только если `tileTemp(фермы) > 0`. Иначе ноль.
- **Множитель холода:** выработка домножается на коэффициент от `cold` колониста.
- **Tailoring:** на `tailor`-верстаке работа `tailor` тратит `wood` и копит прогресс; на пороге — `stock.clothing += 1`. Навык `tailoring`? Используем существующий — в Фазе 0 навыков 8, среди них нет tailoring; добавим переиспользование `cooking`? Нет. **Решение:** одежду шьёт навык `crafting`? Его нет. Чтобы не плодить навыки, в Фазе 1 пошив использует навык `building` (верстак — «ремесло»). Можно добавить отдельный навык позже.
- **Топливо обогрева** — в `temperatureSystem` (см. 4.3), не в `work`.

### 4.6. Порча еды (`foodSpoilage`, в `onNewDay`)
- На каждый новый день: `food.amount *= (1 - spoilFraction)`.
- `spoilFraction = BASE_SPOIL * storageFactor * tempFactor`:
  - `storageFactor`: меньше при большем числе построенных складов (напр. 1 / (1 + builtStorages*0.6)).
  - `tempFactor`: 1 при тепле, →~0.15 при морозе (зимой еда почти не портится — зима убивает производство, но консервирует запас).

## 5. Новые здания (данные + стоимость)

| Тип | Назначение | Стоимость (черновая) | Стройка | Слоты | Проходимо |
|-----|-----------|----------------------|---------|-------|-----------|

| `heater` | Греет комнату, жжёт дерево | wood 30 | 25 | 0 | да |

## 6. Изменения HUD (React)

- Верхний бар: сезон + значок + уличная °C + прогноз (дни до зимы).
- Оверлей температуры (тоггл): тайлы тонируются по тёплый↔холодный.
- Индикатор: запас дерева ↔ расход на обогрев (сколько обогревателей активно).
- Счётчик `stock.clothing` + кнопка приоритета `tailor`.
- Инспектор колониста: добавить полосу «холод» и значок «одет».

## 7. Критерии приёмки

- Замкнутая стенами+дверью комната с работающим обогревателем теплее улицы; пролом стены роняет её температуру к уличной.
- Обогреватель без дерева не греет (комната стынет).
- Зимой уличная ферма даёт 0 еды; ферма в тёплой комнате — продолжает.
- Колонист на морозе без одежды набирает `cold`, работает медленнее и при сильном морозе теряет здоровье; одежда/тёплая комната это снимают.
- Одежда шьётся на верстаке (тратит дерево, растит `stock.clothing`); мёрзнущий колонист авто-одевается.
- Еда вне склада в тепле портится; склад и зимний холод резко замедляют порчу.

## 8. Тест-стратегия (TDD, vitest)

- `roomSystem`: замкнутая коробка → 1 комната; пролом → улица; площадь корректна; пересчёт по изменению стен.
- `temperatureSystem`: обогреватель греет комнату выше улицы; нет дерева → не греет; большая комната греется слабее; уличные тайлы = outdoorTemp.
- `seasonSystem`: переход сезонов по `SEASON_LENGTH`; уличная °C по сезону/фазе; зимний минимум.
- `needs.cold`: рост на морозе/падение в тепле; одежда повышает эфф. температуру; авто-одевание; урон здоровью при замерзании; множитель замедления.
- `work` (зима): ферма на морозном тайле = 0; в тёплой = производит. Tailoring: тратит дерево, растит `stock.clothing`.
- `foodSpoilage`: порча вне склада в тепле; сохранность в складе/в мороз.
- Детерминизм: один сид → идентичный прогон с сезонами.

## 10. Риски и снижение

- **Производительность температуры:** по-комнатная модель — O(числа комнат) на тик, не O(тайлов). `roomSystem` ленив (только при изменении стен). Дёшево.
- **Корректность flood-fill комнат:** покрыто юнит-тестами (замкнутость, пролом, площадь).
- **Баланс зимы (смертельность):** числа черновые, отдельный баланс-проход; цель — зима трудна, но переживаема при подготовке.
- **Совместимость сейвов:** `payloadVersion` 4; старые отвергаются (как уже устроено).
